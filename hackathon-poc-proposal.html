아이디어명

함께 말하고, 함께 회복하는 마음안전 네비게이터

시민이 일상에서 겪는 대화 불안과 감정 과부하 상황을, 한 화면에서 안내·연습·안정까지 연결해 스스로 회복 경로를 찾도록 돕는 체감형 공공 서비스 PoC입니다.

아이디어 요약

문제정의-개발동기-해결방식

문제 정의 감정조절, 사회적 대화 연습, 안정 루틴이 분리되어 있어 시민이 실제 상황에서 즉시 활용하기 어렵습니다.
개발 동기 정책·교육 현장에서 강조되는 정신건강 예방적 접근을, 시민이 스스로 체감할 수 있는 인터랙션으로 구현하고자 했습니다.

기획배경

정책 방향과 시민 경험을 연결하는 디지털 실험

  • 미래사회에서 정서적 회복탄력성은 개인 역량을 넘어 공공 서비스 품질의 핵심 지표가 되고 있습니다.
  • 시민은 “지금 당장 쓸 수 있는” 쉬운 도구를 원하지만, 기존 서비스는 기능 단절로 사용 지속성이 낮습니다.
  • 해당 PoC는 과학기술을 통해 시민이 스스로 문제를 인식하고 행동을 연습하는 참여형 경험을 목표로 합니다.

6 현장 개발 계획

대회 기간 내 구현 범위·R&R·MVP 정의

w 대회 기간 내 구현 범위 명시
  • 모바일 단일 화면 기준으로 홈/타로/사회성/감정선택/안정 모드 전환 UX 완성
  • 자연어 명령(예: 성인, 1번, 랜덤 상황) 기반 시나리오 이동 및 대화 루프 구현
  • 텍스트/음성 입력 공통 파이프라인과 시연 안정화(핵심 오류 처리 포함)
w 팀원별 역할 분담(R&R)
  • 기획·정책 연계: 시민경험 시나리오 설계, 정책 메시지 정합성 검토
  • AI/대화 설계: 프롬프트 구조, 페르소나 제어, 안전 가드레일 설정
  • 프론트엔드 개발: UI 구현, 전환 애니메이션, 툴 라우팅 및 입력 처리
  • 테스트·발표 운영: 통합 점검, 데모 시나리오 리허설, 본선 시연 진행
w 우선 구현 기능(MVP) 정의
  • MVP-1: 사회성 연습(연령 선택 → 상황 선택 → 페르소나 대화 → 피드백)
  • MVP-2: 감정선택(CBT) 및 안정 모드(호흡/명상/사운드) 즉시 진입
  • MVP-3: 하단 대화창 고정 + 자연어 기반 툴 전환
  • MVP-4: 청소년/성인 대표 시나리오 각 1종 이상 본선 데모 패키지 완성
4 AI 모델 및 기술 스택

AI 알고리즘·개발 환경·데이터셋 활용

w 사용할 AI 알고리즘
  • LLM: 사용자 발화 이해, 상황별 대화 생성, 피드백 생성
  • 음성 처리 모델(STT): 음성 입력을 텍스트로 전사하여 동일한 대화 파이프라인에 연결
  • 규칙 기반 라우팅: “성인 1번”, “사회성 연습” 등의 자연어 명령을 시나리오 선택으로 매핑
w 프레임워크 및 개발 환경
  • 프론트엔드: HTML/CSS/Vanilla JavaScript (모바일 우선 단일 페이지 UI)
  • 음성 처리: Web Audio API, MediaRecorder
  • 캐릭터 렌더링: lottie-web + SVG 폴백
  • 백엔드(옵션): Python Flask 프록시 (API 연계 안정화)
  • 개발 환경: 로컬 브라우저 기반 PoC, API 키 기반 외부 모델 연동
w 데이터셋 출처 및 활용 방식
  • 서비스 내 시나리오/대화 데이터: 팀이 기획한 사회적 상황 템플릿 + 런타임 생성 대화 로그
  • 타로 데이터: 공개 API(tarotapi.dev) 카드 메타데이터 + 로컬 카드 이미지 리소스
  • 활용 방식: 세션 단위 맥락 메모리(사용자 최근 발화, 턴 히스토리)로 응답 일관성 강화
  • 개인정보 최소화: PoC 단계에서는 민감정보 저장을 지양하고, 즉시 상호작용 중심으로 운용
5 시스템 아키텍처

전체 흐름 및 AI/인간 판단 지점

w End-to-End 흐름 데이터 수집 (텍스트/음성 입력) → 전처리 (음성 전사, 명령 파싱, 세션 컨텍스트 정리) → 모델 학습 (해커톤 PoC에서는 별도 재학습 없이 사전학습 모델 활용) → 추론 (LLM 응답 생성/스트리밍) → 서비스 UI (메인/타로/사회성/CBT/안정 모드 화면 반영)
w AI 개입 지점
  • 의도 해석 보조: 사용자 자연어를 상황/도구 흐름으로 변환
  • 대화 생성: 페르소나 응답, 피드백 문구, 정서 지원 문장 생성
  • 음성 전사: 음성 입력을 텍스트로 변환해 동일 UX 제공
w 인간 판단 지점
  • 시나리오/정책 문구 설계: 어떤 상황을 어떤 어조로 제시할지 기획자가 통제
  • 안전 가드레일: 부적절 응답 필터링 기준 및 운영 정책 수립
  • 최종 서비스 판단: 시민 피드백 기반으로 기능 채택/개선 우선순위 결정
기대효과

사회적 효과·정책 시사점·시민 인식 변화

  • 사회적 효과: 정서적 어려움 대응을 “치료 이후”가 아닌 “일상 예방” 관점으로 확장
  • 정책적 시사점: 공공기관이 제공할 수 있는 시민 체감형 AI 정서지원 서비스 모델 제시
  • 시민 인식 변화: 감정관리/대화연습을 특별한 일이 아닌 생활 루틴으로 인식하도록 전환
  • 확장 가능성: 과학관 전시, 청소년 프로그램, 지역 정신건강 연계 서비스로의 발전 기반 확보
